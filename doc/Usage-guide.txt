# Описание
Сервер представляет собой Java-server предоставляющий мост между CTI сервером и операторами, предоставляет возможности для логирования и централизованного управления операторами.

# Конфигурация
Для запуска необходим минимальный конфигурационный файл(config.properties):
`
clientgate.server.port=32562

cti.HostA=
cti.PortA=
cti.HostB=
cti.PortB=

webgate.ip.filter=deny
webgate.ip.allow=
webgate.ip.disallow=

events.db.url=
events.db.user=
events.db.password=
`
- *clientgate.server.port* - порт для подключения клиентского приложения операторов
- *cti.HostA;cti.PortA;cti.HostB;cti.PortB* - настройки подключения к CTI серверу
- *webgate.ip.filter* - способ фильтрации запросов к web интерфейсу.
    Доступны два значения:
    - *allow* - разрешает соединения со всех ip, кроме тех что указанны в *webgate.ip.disallow*
    - *deny* - запрещает все соединения со всех ip, кроме тех что указанны в *webgate.ip.allow*
- *events.db.url* - url для подключения к базе данных логирования событий. Напрмиер:127.0.0.1/statistic?charset=utf8
- *events.db.user* - логин для подключания к серверу базы данных
- *events.db.password* - пароль для подключания к серверу базы данных
- есть еще параметры, они будут добавленны позже

# Веб-сервер
- /statistic - выдает статистику работы по серверу - аптайм, подключенные агенты, используемая память, количество запросов к веб и пакетов.
- /make_call/{username}/{number} - инициирует звонок для оператора {username} на телефонный номер {number}
- Если запрос не обрабатывается, то проверяется наличие файла в директории ./www
